管理后台概览

管理后台入口为 /platform/#/admin。用户需要拥有管理员角色，或通过 RBAC 获得相应管理接口权限后才能访问对应页面。

初始管理员​

平台首次启动且管理员组中没有用户时，会自动创建 admin 用户，并把初始密码打印到日志：

Init Admin: Admin{ name: admin, password: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx, email: admin@0rays.club}

Helm 部署查看方式：

kubectl logs -n cbctf deployment/cbctf | grep "Init Admin"

首次登录后请立即修改密码。

常用管理流程​

  1. 登录后台，先在系统状态页确认数据库、Redis、任务队列和运行状态正常。
  2. 在用户、分组、角色权限中创建管理员、主办方或普通用户，并按需分配权限。
  3. 在题目管理中创建题目，配置附件、Flag、动态生成器或容器靶机。
  4. 对容器题使用测试启动，确认镜像、端口、Flag 注入和访问地址正常。
  5. 在比赛管理中创建比赛，设置时间、队伍人数、规则、奖项、公告和题目。
  6. 比赛前按需预热镜像、生成器或靶机。
  7. 比赛期间查看队伍、提交、排行榜、公告、作弊记录、任务和运行中的靶机。
  8. 比赛结束后下载 Writeup、流量文件或附件，并清理不再需要的靶机和文件。

角色和权限​

系统初始化默认角色：

角色说明
admin系统管理员，拥有全部权限
organizer赛事主办方，拥有比赛、题目、队伍、公告、作弊、比赛内靶机等管理权限
user普通选手，拥有参赛、队伍、题目、提交、靶机控制和 Writeup 权限

后台导航会根据当前用户可访问接口过滤。授权后如果页面仍不可见，重新登录或检查 /me/permissions 返回的权限列表。

注意事项​

  • 系统配置在线更新适合调整运行参数；生产部署中的域名、密钥、数据库和 Redis 建议仍由 Helm values 或 Secret 管理。
  • 删除文件、停止靶机、删除比赛或题目前请确认是否仍有比赛数据需要保留。
  • 容器题上线前建议完成一次测试启动和停止，避免比赛期间暴露镜像、端口或资源配置问题。
